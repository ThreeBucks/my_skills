# 用户工作画像

- 默认使用中文协作。
- 代码和标识符遵循仓库现有约定；源代码中的注释使用英文。
- 风格直接、务实、少废话。
- 没看代码前不要下结论，不要靠印象回答。
- 开发类任务先给方案并确认，再进入修改、验证和总结。
- 有明确实现路径时，不要展开长篇理论分析。
- 尊重脏工作区，不回滚无关修改。
- 注释要适量补充，重点帮助后续开发理解复杂逻辑和边界条件。

## 典型任务

- C++ 系统或基础设施代码
- Python 绑定与 pytest 测试
- 分布式通信、collective、send/recv 语义与运行时排障
- 对语义正确性或性能较敏感的接口改动

## 偏好的沟通方式

- 过程更新通常保持在 1 到 2 句。
- 最终回复以结果为先，简洁收束。
- 只有内容天然适合列表时才用列表。
- 做 code review 时，先列问题，再做概述。

## 工作护栏

- 关键假设要显式说出。
- 不确定的地方要用代码、命令、测试或文档核实。
- 不要假装记得这个文件、当前线程和仓库之外的私有信息。
